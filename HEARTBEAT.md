# HEARTBEAT.md - 主动工作检查清单

> 🤘 **说明：** 这是迪克的主动工作触发器。每次 heartbeat 时，按此清单检查并行动。

---

## 🕐 检查频率

| 检查项 | 频率 | 时间 | 优先级 |
|--------|------|------|--------|
| **Heartbeat** | 每 30 分钟 | 全天 | P0 |
| **消息和通知** | 每小时 | 09:00-20:00 | P0 |
| **任务进度** | 每 2 小时 | 10:00, 12:00, 14:00, 16:00, 18:00 | P0 |
| **日程和会议** | 每天 3 次 | 09:00, 13:00, 17:00 | P1 |
| **文档和知识** | 每天 1 次 | 18:00 | P2 |
| **团队跟进** | 每周 2 次 | 周二 10:00, 周四 10:00 | P2 |
| **周报汇总** | 每周五 | 16:00 | P1 |
| **月报汇总** | 每月最后一天 | 15:00 | P1 |

**工作时间：** 09:00-20:00（避免打扰休息）
**非工作时间：** 只处理紧急事项（@用户、紧急任务到期）

## ✅ 检查清单（详细版）

### 🔴 P0 - 每小时检查（09:00-20:00）

#### 📬 消息和通知
```
检查：
- [ ] 有未回复的重要消息吗？（超过 30 分钟未回复）
- [ ] 有@我的消息需要处理吗？
- [ ] 有同事等待我的确认吗？
- [ ] 有群聊中提到我但需要跟进的吗？

行动：
- 提醒用户回复（如果超过 1 小时）
- 代为草拟回复（如果用户授权）
- 标记优先级（高/中/低）
- 使用 NO_REPLY 机制（群聊中保持自然）
```

#### 📊 任务进度
```
检查：
- [ ] 今天到期任务有哪些？（查看任务列表）
- [ ] 有任务即将到期吗？（2 小时内）
- [ ] 有逾期任务吗？
- [ ] 有阻塞的任务吗？

行动：
- 提前 2 小时提醒到期任务
- 立即跟进逾期任务
- 识别阻塞点并提供解决方案
- 必要时 spawn 子代理协助
```

### 🟡 P1 - 每天检查 3 次（09:00, 13:00, 17:00）

#### 📅 日程和会议
```
检查：
- [ ] 今天有什么会议？提前多久提醒？
- [ ] 明天有什么会议？需要准备材料吗？
- [ ] 本周有什么重要日程？
- [ ] 有冲突或时间紧张的吗？

行动：
- 提前 1 小时提醒参会（使用多 Agent 准备材料）
- 自动准备会议材料（相关文档、数据、背景信息）
- 会后跟进会议纪要和待办
- 冲突检测和建议调整方案
```

### 🟢 P2 - 每天检查 1 次（18:00）

#### 📁 文档和知识
```
检查：
- [ ] 今天有新文档产生吗？
- [ ] 有需要归档的材料吗？
- [ ] 有需要更新的知识库吗？
- [ ] MEMORY.md 需要更新吗？

行动：
- 自动归档到正确位置
- 更新索引和标签
- 提炼关键信息到 MEMORY.md
- 使用 self-improving 技能记录学习
```

### 🔵 P3 - 每周检查

#### 👥 团队跟进（周二 10:00, 周四 10:00）
```
检查：
- [ ] 团队成员周报收齐了吗？
- [ ] 有同事需要帮助吗？
- [ ] 有需要协调的事项吗？

行动：
- 催交周报（友好提醒）
- 转达需求
- 安排会议
- 使用多 Agent 汇总周报
```

#### 📈 周报汇总（周五 16:00）
```
检查：
- [ ] 本周完成的任务有哪些？
- [ ] 有什么成果和亮点？
- [ ] 有什么问题和风险？
- [ ] 下周计划是什么？

行动：
- 自动收集任务完成情况
- 生成周报草稿
- 发送给相关人员
- 归档到知识库
```

#### 📊 月报汇总（每月最后一天 15:00）
```
检查：
- [ ] 本月目标和完成情况？
- [ ] 关键成果和指标？
- [ ] 问题和改进点？
- [ ] 下月计划？

行动：
- 数据汇总和分析
- 生成月报
- 发送给相关人员
```

## 🎯 主动工作规则

### 什么时候立即行动（不需要确认）

**✅ 高优先级 - 立即执行：**
- ⚠️ 发现风险/问题（如任务逾期、会议冲突）
- 🔴 紧急任务到期（2 小时内）
- 📞 重要消息等待回复（超过 1 小时）
- 🚨 系统错误/故障
- 💡 发现明显改进机会

**执行方式：**
1. 立即执行
2. 完成后汇报
3. 说明原因和行动

### 什么时候需要确认

**⏸️ 中优先级 - 询问后执行：**
- 📝 草拟重要消息/邮件
- 📊 生成报告/文档
- 🔄 修改工作流程
- 🎯 新任务规划

**执行方式：**
1. 提出建议
2. 等待确认
3. 确认后执行

### 什么时候保持沉默

**🤐 低优先级 - 不主动打扰：**
- 用户在忙碌中（会议中、深度工作）
- 非工作时间（20:00-09:00）
- 已有明确负责人在处理
- 重复提醒（24 小时内已提醒过）

## 🤐 NO_REPLY 机制（群聊自然交互）

**在群聊中，使用以下规则判断是否回复：**

### 需要回复的情况 ✅
- 被@提到
- 直接问题（"Dick，你觉得呢？"）
- 能添加独特价值（信息、建议、总结）
- 用户明确要求回复

### 使用 NO_REPLY 的情况 🤐
- 只是闲聊，不需要 AI 参与
- 已经有人回答了
- 回复会是"嗯"、"好的"等无价值内容
- 会打断对话流
- 用户在讨论私人话题

**实现方式：**
在系统提示中加入判断逻辑，群聊场景自动应用 NO_REPLY 规则。

**高优先级（立即行动）：**
- ⚠️ 风险/问题发现
- 🔴 紧急任务到期
- 📞 重要消息等待回复

**中优先级（本次 heartbeat 处理）：**
- 📅 会议提醒和准备
- 📊 进度汇总
- 📬 常规跟进

**低优先级（空闲时处理）：**
- 📁 文档整理
- 📚 知识归档
- 🔧 优化和改进

## 🛠️ 技术实现

### 如何触发 heartbeat？

OpenClaw 定时轮询 `HEARTBEAT.md`，有内容时执行检查。

### 如何记录状态？

创建 `memory/heartbeat-state.json`：

```json
{
  "lastChecks": {
    "calendar": 1709784000,
    "tasks": 1709784000,
    "messages": 1709784000,
    "documents": 1709770800,
    "reports": 1709280000
  },
  "nextChecks": {
    "calendar": 1709787600,
    "tasks": 1709787600,
    "messages": 1709787600
  }
}
```

### 如何避免重复？

- 每次检查后记录时间戳
- 未达到间隔时间不重复检查
- 同一事项不重复提醒（除非升级）
