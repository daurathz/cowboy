# HEARTBEAT.md - 主动工作检查清单

> 🤘 **说明：** 这是迪克的主动工作触发器。每次 heartbeat 时，按此清单检查并行动。

---

## 🕐 检查频率

- **默认：** 每 30-60 分钟自动检查一次
- **工作时间：** 09:00-20:00（避免打扰休息）
- **可调整：** 根据实际情况修改

---

## ✅ 检查清单

### 📅 日程和会议（每天检查 3 次）

```
时间：09:00, 13:00, 17:00

检查：
- [ ] 今天有什么会议？提前多久提醒？
- [ ] 明天有什么会议？需要准备材料吗？
- [ ] 本周有什么重要日程？
- [ ] 有冲突或时间紧张的吗？

行动：
- 提前 1 小时提醒参会
- 自动准备会议材料（相关文档、数据）
- 会后跟进会议纪要和待办
```

### 📬 消息和通知（每小时检查）

```
检查：
- [ ] 有未回复的重要消息吗？
- [ ] 有@我的消息需要处理吗？
- [ ] 有同事等待我的确认吗？

行动：
- 提醒用户回复
- 代为草拟回复
- 标记优先级
```

### 📊 任务和进度（每天检查 2 次）

```
时间：10:00, 16:00

检查：
- [ ] 今天到期任务有哪些？
- [ ] 逾期任务有哪些？
- [ ] 团队成员任务完成情况？
- [ ] 有风险或延迟的吗？

行动：
- 提醒到期任务
- 跟进逾期任务
- 汇总进度并汇报
```

### 📁 文档和知识（每天检查 1 次）

```
时间：18:00

检查：
- [ ] 今天有新文档产生吗？
- [ ] 有需要归档的材料吗？
- [ ] 有需要更新的知识库吗？

行动：
- 自动归档到正确位置
- 更新索引和标签
- 提炼关键信息到 MEMORY.md
```

### 📈 数据和报表（按周期）

```
周报：每周五 16:00
月报：每月最后一天 15:00

检查：
- [ ] 需要收集哪些数据？
- [ ] 需要汇总哪些内容？
- [ ] 需要发送给谁？

行动：
- 自动拉取数据
- 生成报表
- 发送给相关人员
```

### 👥 团队跟进（每周检查 2 次）

```
时间：周二 10:00, 周四 10:00

检查：
- [ ] 团队成员周报收齐了吗？
- [ ] 有同事需要帮助吗？
- [ ] 有需要协调的事项吗？

行动：
- 催交周报
- 转达需求
- 安排会议
```

## 🎯 优先级规则

**高优先级（立即行动）：**
- ⚠️ 风险/问题发现
- 🔴 紧急任务到期
- 📞 重要消息等待回复

**中优先级（本次 heartbeat 处理）：**
- 📅 会议提醒和准备
- 📊 进度汇总
- 📬 常规跟进

**低优先级（空闲时处理）：**
- 📁 文档整理
- 📚 知识归档
- 🔧 优化和改进

## 🛠️ 技术实现

### 如何触发 heartbeat？

OpenClaw 定时轮询 `HEARTBEAT.md`，有内容时执行检查。

### 如何记录状态？

创建 `memory/heartbeat-state.json`：

```json
{
  "lastChecks": {
    "calendar": 1709784000,
    "tasks": 1709784000,
    "messages": 1709784000,
    "documents": 1709770800,
    "reports": 1709280000
  },
  "nextChecks": {
    "calendar": 1709787600,
    "tasks": 1709787600,
    "messages": 1709787600
  }
}
```

### 如何避免重复？

- 每次检查后记录时间戳
- 未达到间隔时间不重复检查
- 同一事项不重复提醒（除非升级）
