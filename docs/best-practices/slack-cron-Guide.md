## 📋 slack-cron 能做什么

**slack-cron** 是一个基于 OpenClaw 的定时提醒工具，可以通过 Slack 发送定时消息。

### 核心能力

| 功能 | 说明 | 示例 |
|------|------|------|
| ⏰ 一次性提醒 | 在指定时间提醒一次 | "5 分钟后提醒我喝水" |
| 🔄 周期提醒 | 按 cron 表达式重复提醒 | "每天早上 8 点提醒我打卡" |
| 📅 工作日提醒 | 仅工作日触发 | "工作日下午 6 点提醒我写日报" |
| 🌐 多渠道支持 | 支持 Slack/QQ/钉钉/企微 | 配置后自动适配 |
| 🗑️ 自动清理 | 一次性任务执行后自动删除 | 无需手动管理 |

## 💬 如何使用

### 直接用自然语言说

**对 Dick 说：**

```
"5 分钟后提醒我喝水"
"每天早上 8 点提醒我打卡"
"工作日下午 6 点提醒我写日报"
"每小时提醒我休息一下"
"每周一上午 9 点提醒我开例会"
"7 点到 21 点每个整点提醒我喝水"
```

### 技术实现

**底层调用：**
```bash
curl -X POST http://localhost:10260/cron \
  -H "Content-Type: application/json" \
  -d '{
    "action": "add",
    "job": {
      "name": "提醒名称",
      "schedule": {
        "kind": "at",
        "atMs": 1773053871000
      },
      "payload": {
        "kind": "agentTurn",
        "message": "提醒内容",
        "deliver": true,
        "channel": "slack",
        "to": "U05AN0GGM3M"
      },
      "deleteAfterRun": true
    }
  }'
```

## 📚 使用场景

### 场景 1：喝水提醒

**需求：** 每天 7:00-21:00，每个整点提醒喝水

**说法：**
> "每天早上 7 点到晚上 9 点，每个整点提醒我喝水"

**Cron 表达式：** `0 7-21 * * *`

---

### 场景 2：写日报

**需求：** 工作日下午 6 点提醒写日报

**说法：**
> "工作日下午 6 点提醒我写日报"

**Cron 表达式：** `0 18 * * 1-5`

---

### 场景 3：会议提醒

**需求：** 每周一上午 9 点开例会

**说法：**
> "每周一上午 9 点提醒我开例会"

**Cron 表达式：** `0 9 * * 1`

---

### 场景 4：休息提醒

**需求：** 每小时休息 5 分钟

**说法：**
> "每小时提醒我休息一下"

**Cron 表达式：** `0 * * * *`

---

### 场景 5：一次性提醒

**需求：** 10 分钟后提醒我关火

**说法：**
> "10 分钟后提醒我关火"

**类型：** `at`（绝对时间戳）

## 📊 任务管理

### 查询任务

**说法：**
> "我有哪些提醒？"

**底层调用：**
```bash
curl http://localhost:10260/cron -X POST \
  -H "Content-Type: application/json" \
  -d '{"action":"list"}'
```

---

### 删除任务

**说法：**
> "取消喝水提醒"

**底层调用：**
```bash
curl http://localhost:10260/cron -X POST \
  -H "Content-Type: application/json" \
  -d '{"action":"remove","jobId":"任务 ID"}'
```

---

### 暂停/恢复任务

**说法：**
> "暂停打卡提醒"
> "恢复打卡提醒"

## 📝 Cron 表达式参考

| 表达式 | 说明 |
|--------|------|
| `0 8 * * *` | 每天早上 8 点 |
| `0 22 * * *` | 每天晚上 10 点 |
| `0 9 * * 1-5` | 工作日早上 9 点 |
| `0 9 * * 1` | 每周一早上 9 点 |
| `0 10 * * 0,6` | 每周末上午 10 点 |
| `0 * * * *` | 每小时整点 |
| `*/30 * * * *` | 每 30 分钟 |

## ⚠️ 注意事项

1. **时区：** 默认 Asia/Shanghai
2. **消息权限：** Slack Bot 需要有主动消息权限
3. **任务存储：** JSON 文件持久化 (`~/.openclaw/cron/jobs.json`)
4. **调度精度：** 每分钟检查一次
5. **一次性任务：** 执行后自动删除

---

## 🛠️ 故障排查

### 问题：消息没收到

**检查：**
1. cron 工具是否在运行：`curl http://localhost:10260/health`
2. 日志是否有错误：`tail /tmp/cron-tool.log`
3. Slack Bot 权限是否正常

### 问题：任务没执行

**检查：**
1. 任务是否启用：`curl http://localhost:10260/cron -d '{"action":"list"}'`
2. 调度器是否运行：查看日志 `[CronScheduler]` 相关输出
3. 时间是否正确：检查 `next_run_at` 字段
