## 1. 快速入门

### 1.1 什么是 OpenClaw？

OpenClaw 是一个 AI 助手框架，可以接入钉钉、微信、Telegram 等多个渠道。我们的 AI 助手叫 **迪克 (Dick)** 🤘，带着摇滚灵魂，能帮你：

- 📄 处理文档（Word、Excel、PDF）
- 💻 编写和审查代码
- 📊 数据分析和报表
- 🔍 信息检索和总结
- 📅 提醒和任务管理
- 🗣️ 日常对话和问答

### 1.2 如何开始对话？

1. **打开钉钉**，找到迪克的对话框
2. **直接发消息**，就像和同事聊天一样
3. **发送文件**，点击"文件"按钮或拖拽上传

### 1.3 第一句话怎么说？

```
你好！/ 嘿！/ 在吗？
```

迪克会自动回应，然后你就可以说需求了！

## 2. 常见问题解答

### 2.1 文件相关

**Q: 怎么发送文件给迪克？**

A: 三种方式：
- 点击钉钉对话框的"文件"按钮 📎
- 直接拖拽文件到对话框
- 复制粘贴截图/图片

**Q: 迪克能处理哪些文件类型？**

A: 
| 类型 | 支持格式 | 能做什么 |
|------|---------|---------|
| 文档 | doc, docx, pdf, txt, md | 读取、总结、改写、翻译 |
| 表格 | xls, xlsx, csv | 数据分析、统计、图表 |
| 图片 | jpg, png, gif, webp | 文字识别、内容分析 |
| 代码 | js, py, sh, json, yaml | 审查、调试、优化 |

**Q: 文件太大怎么办？**

A: 
- 单个文件建议 < 10MB
- 超大文件可以分开发送
- 或者告诉迪克"只处理前 100 行"

**Q: 迪克能发文件给我吗？**

A: 可以！直接说：
```
"把 XXX 文件发给我"
"把刚才的报告发我"
```

---

### 2.2 对话相关

**Q: 迪克回复太慢了怎么办？**

A: 
- 复杂任务需要思考时间（正常 10-30 秒）
- 可以说"先给个大概"让它快速回应
- 避免同时问多个问题

**Q: 迪克理解错了我的意思**

A: 
- 直接说"不对，我意思是..."
- 或者"重新来，我要的是..."
- 迪克不会生气，越说越明白

**Q: 对话太长迪克会忘吗？**

A: 
- 迪克有记忆，但太长可能遗漏
- 重要信息可以说"记住这个：..."
- 新话题可以说"换个话题，..."

---

### 2.3 能力边界

**Q: 迪克能做什么？**

A: 
- ✅ 读取和分析你发送的内容
- ✅ 处理工作区内的文件
- ✅ 搜索网络信息
- ✅ 编写代码和文档
- ✅ 数据分析和总结
- ✅ 发送文件给你

**Q: 迪克不能做什么？**

A: 
- ❌ 访问你的私人文件（除非你发送）
- ❌ 代表你发送邮件/消息（需要确认）
- ❌ 执行危险操作（删除、转账等）
- ❌ 预测未来或给投资建议
- ❌ 访问公司内部系统（除非配置）

## 3. 提示词技巧

### 3.1 基本原则

**✅ 好的提示词：**
```
"帮我分析这个 Excel，统计每个城市有多少设备"
"把这份文档总结成 3 个要点"
"检查这段代码有没有 bug"
```

**❌ 模糊的提示词：**
```
"看看这个"
"处理一下"
"搞快点"
```

### 3.2 场景化示例

#### 📊 数据分析

```
❌ "分析这个表格"

✅ "分析这个 Excel，告诉我：
    1. 总共有多少条记录
    2. 每个城市各有多少设备
    3. 哪些设备维保快到期了（30 天内）"
```

#### 📝 文档处理

```
❌ "总结一下"

✅ "把这份文档总结成：
    - 3 个核心观点
    - 5 个关键数据
    - 需要跟进的事项列表"
```

#### 💻 代码相关

```
❌ "这段代码有问题"

✅ "检查这段 Python 代码：
    1. 有没有语法错误
    2. 有没有性能问题
    3. 给出优化建议
    4. 直接给出修改后的代码"
```

#### 🔍 信息检索

```
❌ "查查这个"

✅ "帮我搜索：
    - 主题：OpenClaw 钉钉配置
    - 找最新的官方文档
    - 只要中文结果
    - 列出前 5 个链接和摘要"
```

### 3.3 分步引导

复杂任务可以分步说：

```
第一步："先看看这个文件有多少行，有哪些列"
第二步："好，现在帮我统计每列的空值数量"
第三步："最后生成一个数据质量报告"
```

### 3.4 设定角色

```
"你现在是数据分析师，帮我..."
"假设你是资深程序员，审查这段代码..."
"用产品经理的视角，评估这个需求..."
```

### 3.5 指定格式

```
"用表格形式输出"
"用 Markdown 格式"
"用 bullet points 列出"
"输出 JSON 格式"
```

## 4. 避免死循环

### 4.1 什么是死循环？

迪克有时会钻牛角尖，比如：
- 反复尝试同一个方法
- 纠结于细节无法推进
- 卡在某一步一直重试

### 4.2 如何识别？

迪克出现以下情况可能陷入死循环：
- 同样的话说了 3 遍以上
- 一直在"让我试试..."但没有结果
- 超过 2 分钟没有实质性进展

### 4.3 如何打断？

**方法 1：直接打断**
```
"停，换个方法"
"先别管这个，我们..."
"跳过这一步，直接..."
```

**方法 2：重置话题**
```
"我们重新来，我要的是..."
"忘掉刚才的，新任务是..."
```

**方法 3：简化任务**
```
"先给个简单版本"
"不用太复杂，只要..."
"大概就行，不需要精确"
```

### 4.4 预防方法

**✅ 明确终止条件**
```
"如果找不到，就告诉我没有"
"最多试 3 次，不行就放弃"
"10 分钟内给不出结果就提醒我"
```

**✅ 设定边界**
```
"只需要前 100 行数据"
"只看最近 3 个月的"
"预算不超过 1000 元"
```

**✅ 分阶段确认**
```
"先做第一步，完成后等我确认"
"每完成一步告诉我一声"
"有问题随时问我"
```

## 5. 最佳实践场景

### 5.1 晨会准备

```
"帮我准备晨会材料：
1. 昨天完成了什么
2. 今天计划做什么
3. 有什么风险需要关注
从我的任务列表里提取"
```

### 5.2 周报生成

```
"根据我这周的工作记录，生成周报：
- 格式：Markdown
- 包含：完成事项、数据指标、下周计划
- 语气：专业但轻松
- 长度：500 字左右"
```

### 5.3 会议纪要

```
"这是会议录音转文字，帮我：
1. 提取关键决策
2. 列出待办事项（含负责人）
3. 总结争议点
4. 生成会议纪要文档"
```

### 5.4 代码审查

```
"审查这段代码，关注：
1. 安全性问题
2. 性能瓶颈
3. 代码规范
4. 给出修改建议
5. 直接给出修复后的代码"
```

### 5.5 数据报表

```
"分析这个销售数据：
1. 本月 vs 上月对比
2. Top 10 产品
3. 异常数据标记
4. 生成可视化图表建议
5. 输出 Excel 格式"
```

### 5.6 文档翻译

```
"翻译这份文档：
- 源语言：英文
- 目标语言：中文
- 保持专业术语准确
- 保留原文格式
- 不确定的术语标注出来"
```

## 6. 故障排查

### 6.1 迪克没反应

**可能原因：**
- 网络问题
- 服务重启中
- 消息太多排队中

**解决方法：**
```
1. 等 1 分钟再试
2. 发个"在吗？"测试
3. 联系管理员检查服务
```

### 6.2 文件发送失败

**可能原因：**
- 文件太大
- 格式不支持
- 网络中断

**解决方法：**
```
1. 检查文件大小（<10MB）
2. 转换成常见格式
3. 重新发送
4. 或者分开发送
```

### 6.3 迪克理解错误

**可能原因：**
- 提示词模糊
- 缺少上下文
- 任务太复杂

**解决方法：**
```
1. 重新描述，更具体
2. 提供背景信息
3. 分解成小任务
4. 举例说明
```

### 6.4 迪克卡住了

**可能原因：**
- 任务超出能力
- 缺少必要信息
- 陷入死循环

**解决方法：**
```
1. 说"停，换个方法"
2. 简化任务要求
3. 提供更多信息
4. 必要时人工介入
```
